# 页面配置

## 说明

页面是层级最高的容器，在导航栏上会出现对应的页面导航。每个页面可包含一个或多个模块。

页面的ID将用于路由匹配，因此是必须的。

## 示例

```json
{
    "pages": [
        {
            "name": "基本配置",
            "id": "base",
            "modules": []
        }
    ]
}
```

## 配置细项

字段名称    | 说明     | 类型  |  默认值  | 必须
:------ | :----- | :----- | :- | :---
desc    | 页面描述   | String | `''`  | 否
modules | 模块ID集合 | Array |  `[]`  | 否
name    | 页面名称   | String | `''`  | 是
id      | 页面的ID  | String | `''`  | 是
